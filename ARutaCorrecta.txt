SISTEMA DE RUTAS Y AUTENTICACIÓN - WISBE (Actualizado)

Este archivo detalla el flujo de navegación y seguridad para asegurar la correcta configuración en wisbe.xyz.

1. FLUJO DE LOGIN (docs/login.html)
   - Al cargar, login.html realiza una verificación doble:
     a) `supabase.auth.getSession()` -> Verifica si hay una sesión activa en el servidor.
     b) `Auth.getUser()` -> Recupera los datos del usuario del localStorage.
   - Solo si AMBAS coinciden, se redirige al panel.
   - Si no hay sesión en el servidor pero sí hay datos locales, se limpia el localStorage (`gym_user`) para evitar el error de "parpadeo" o bucle infinito.

2. DIRECCIONAMIENTO POR ROL/UNIDAD (docs/assets/js/utils.js -> Auth.getRedirectUrl)
   - El sistema decide a qué página enviarte según tu perfil:
     * SI eres Admin Global (gym-admin) o unidad 'wisbe' -> panelWisbe.html
     * SI eres de Belleza (beauty) -> beauty.html
     * SI eres de Tienda (tienda) -> store.html
     * SI eres Independiente (freelance, independiente, construction, school) -> independientes.html
     * CUALQUIER OTRO (gym-owner, etc) -> gym.html (Default)

3. SISTEMA DE LINKS CORTOS / SUBDOMINIOS (docs/404.html)
   - Wisbe utiliza una técnica de "404 routing" para permitir links como `wisbe.xyz/nombreusuario`.
   - Cuando el servidor no encuentra el archivo (ej: /juanito), carga `404.html`.
   - El script en `404.html` analiza la URL:
     * Si la palabra es "RESERVADA" (gym, beauty, store, independientes, etc.), te lleva a la página `.html` del sistema.
     * Si NO es reservada, asume que es un nombre de usuario y te redirige a:
       `cards.html?u=nombreusuario`
   - IMPORTANTE: Para que esto funcione en wisbe.xyz, el archivo `404.html` DEBE estar en la raíz del hosting y el servidor debe estar configurado para usarlo como página de error.

4. PÁGINA DE GESTIÓN (docs/independientes.html)
   - Es el centro de mando para usuarios "freelance" o "independientes".
   - Desde aquí pueden ir a `cards.html` (Editor/Vista) para configurar su tarjeta.
   - El link que se les asigna (wisbe.xyz/nombre) depende enteramente de que su `username` en la base de datos coincida con el link que están probando.

5. VERIFICACIÓN DE ARCHIVOS EN REPOSITORIO:
   - Asegúrate de que todos los archivos dentro de `docs/` estén en la raíz de tu servidor wisbe.xyz (o que el servidor apunte a la carpeta `docs`).
   - El archivo `404.html` es CRÍTICO para los links cortos. Si no está, verás el error 404 estándar del servidor.
